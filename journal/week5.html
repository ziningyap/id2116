This week, we looked into coding our p5.js so that the right video
will appear with their corresponding triggers for the individual key video states.


Our key video states had to include the following functions:
+ Videos appearing and playing
+ Videos hidden and paused
+ Videos at normal speed vs sped up

All whilst following the condition that there is smooth transition from each video state to another through the assigned trigger actions and that only one video would be playing at a time.

Controlling two separate videos concurrently
using two different sensors → setting two separate conditions so they can run concurrently.

State Transition of the Umbrella

We got started on connecting the microbit's collected data and p5.js together through the bluetooth uart service. Our initial plan was to gather brightness and acceleration data from the microbit and translate these numbers to our intended functions on p5.js


However, we kept facing the same microbit error 020 when we tried to read the brightness and acceleration data together :-( Apparently, this was too complex or contained a lot of large variables for the microbit. It is able to write line for a single individual input data though.
