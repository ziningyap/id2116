We plan on using the umbrella's action to play videos. We also wanted the videos to have some visual similarity to the action of the umbrella.
Thus, we came up with the following idea: PLAYING WITH FRUITS!

OPENING: launching of fruits
SPINNING: spinning blade to blend fruits
(+bonus: playing with spinning speed)

State Transition of the Umbrella

We got started on connecting the microbit's collected data and p5.js together through the bluetooth uart service. Our initial plan was to gather brightness and acceleration data from the microbit and translate these numbers to our intended functions on p5.js


However, we kept facing the same microbit error 020 when we tried to read the brightness and acceleration data together :-( Apparently, this was too complex or contained a lot of large variables for the microbit. It is able to write line for a single individual input data though.


Despite this, we decided that it would still be good to make sure that the data values tracked by the microbit was able accurately translated to relevant function commands coded in p5 for the videos. For instance, the controlling of video playback and playback speed by light level of surroundings and acceleration strength respectively.
